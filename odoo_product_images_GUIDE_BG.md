# Как да вземете всички снимки на продукт от Odoo — Ръководство за интеграция

Това ръководство обяснява скрипта **`odoo_product_images_example.py`**: какво прави,
какво е нужно, за да го стартирате, и как разработчиците на сайта да го използват.
Написано е така, че нетехнически читател да следва *какво* и *защо*, с ясно обозначени
секции за разработчиците, които правят самата интеграция.

---

## 1. Накратко: какво е това?

Homekitfit пази **снимките на всеки продукт в Odoo**. Всеки продукт има:

- **една основна снимка** и
- **произволен брой допълнителни снимки** (галерия).

Този скрипт е **мостът**, който позволява на сайта да изтегли **всички снимки на даден
продукт**, в правилния ред, директно от Odoo. Разработчиците го включват в сайта, така
че продуктовите страници винаги да показват актуалните снимки.

> **Къде се пазят снимките (важно):**
> - **Основната снимка** е полето `image_1920` върху продукта (`product.template`).
> - **Всяка друга снимка** е отделен запис в галерията **`cloudcart.product.image`**,
>   свързан с продукта и подреден чрез поле `sequence`.
>
> Тоест **„всички снимки" = основната снимка + записите от галерията, по ред `sequence`.**

---

## 2. Какво прави скриптът (стъпка по стъпка)

1. **Влиза** в Odoo с API ключ.
2. **Намира продукта** по неговото **CloudCart ID**, или **SKU**, или **баркод**.
3. **Прочита основната снимка** (`image_1920`) от продукта.
4. **Прочита галерията** (`cloudcart.product.image`) за този продукт, **подредена**.
5. **Връща/записва всяка снимка по ред** — първо основната, после галерия #2, #3, …

Всяка снимка идва като **данни за изображение** (Base64), които скриптът декодира и
записва като реални `.jpg` / `.png` / `.webp` файлове. Повторното изпълнение е винаги
безопасно — той само чете.

## 4. Как да го стартирате

```bash
export ODOO_USER='website-api'
export ODOO_KEY='<API ключът на този потребител>'

# Всички снимки на продукта с CloudCart id 1040, пълен размер:
python3 odoo_product_images_example.py --cloudcart-id 1040

# По SKU или баркод:
python3 odoo_product_images_example.py --sku ABC123
python3 odoo_product_images_example.py --barcode 3800235261842

# По-малък размер и изходна папка:
python3 odoo_product_images_example.py --cloudcart-id 1040 --size image_512 --out ./downloaded
```

## 5. Справка за разработчици (API заявките)

Скриптът говори с Odoo през **JSON‑RPC** (`POST /jsonrpc`, само вграден Python).
Може да ползвате и **XML‑RPC** (`/xmlrpc/2/...`) по същия начин.

**Стъпка 1 — автентикация** → получавате `uid`:

```
service="common", method="authenticate", args=[db, user, api_key, {}]
```

**Стъпка 2 — намиране на product.template id** (изберете един начин):

```python
# по CloudCart id (пази се като текст; допуснете завършек ".0")
product.template.search_read(
    ['|', ('x_studio_cloudcart_id','=','1040'), ('x_studio_cloudcart_id','=','1040.0')],
    fields=['id','name'], limit=1)

# или по SKU
product.template.search_read([('default_code','=','ABC123')], fields=['id','name'], limit=1)

# или по баркод
product.template.search_read([('barcode','=','3800235261842')], fields=['id','name'], limit=1)
```

**Стъпка 3 — прочитане на основната снимка** от продукта:

```python
product.template.read([template_id], ['image_1920'])
```

**Стъпка 4 — прочитане на галерията, подредена:**

```python
cloudcart.product.image.search_read(
    [('product_tmpl_id','=', template_id)],
    fields=['image_1920','name','sequence','cloudcart_src'],
    order='sequence, id')
```

**Стъпка 5 — сглобяване на подредения списък:** основната `image_1920` е снимка **#1**,
след това записите от галерията по ред `sequence` са #2, #3, … Всяко поле `image_*` е
снимката, кодирана в **Base64**; декодирайте го, за да получите суровите байтове.

### Размери на изображенията (Odoo генерира всички предварително)

| Поле | За какво |
|------|----------|
| `image_1920` | пълен размер (по подразбиране) |
| `image_1024` | голям |
| `image_512` | среден / продуктова страница |
| `image_256` | миниатюра |
| `image_128` | малка миниатюра |

И основната снимка на продукта, **и** всеки запис от галерията предоставят и петте
размера, така че заявявате този, който ви трябва — напр. `image_512` за страницата,
`image_128` за лента с миниатюри.

### Полезни полета в `cloudcart.product.image`

| Поле | Значение |
|------|----------|
| `product_tmpl_id` | продуктът, към който принадлежи снимката |
| `sequence` | ред на показване (по-малко = по-напред) |
| `image_1920` … `image_128` | снимката, във всеки размер (Base64) |
| `name` | етикет на снимката |
| `cloudcart_src` | оригинален CloudCart URL (само за проследяване) |

## 6. Важни бележки

> **Снимките се отдават през API като данни (Base64), а не като публични URL адреси.**
> Директен линк като `/web/image/...` **няма** да работи за анонимни посетители — Odoo
> връща заместващо изображение (placeholder), защото тези снимки не са публикувани в
> публичен сайт. Правилният подход е: вашият бекенд **изтегля снимките през API** (както
> е показано тук) и ги **кешира/отдава от ваша страна** (вашето CDN, хранилище или хост
> за `<img>`). Изтеглете веднъж и кеширайте; не викайте Odoo при всяко зареждане.

- **Използвайте отделен API потребител** с права за четене — никога лична парола.
- **Форматите варират** (`.jpg`, `.png`, `.webp`); разпознайте по байтовете (скриптът го прави).
- **Редът е важен** — винаги сортирайте галерията по `sequence`.
- **Само четене и безопасно** — никога не записва в Odoo; изпълнявайте колкото пъти е нужно.
